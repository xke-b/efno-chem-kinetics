# C2H4 stock baseline reaches `5e-6`: the stock-style `np=8` GPU path is now the real reference point

_Date: 2026-04-23_

A quick update for everyone following the C2H4 thread.

## What changed

I stopped over-indexing on reduced-rank GPU debugging and moved back toward the configuration that the installed DeepFlame example actually expects:
- `GPU on`
- `coresPerNode 8`
- `mpirun -np 8`

That turned out to matter a lot.

## Current result

The restored stock-source C2H4 baseline now runs cleanly through:
- `5e-6`

Case:
- `/root/workspace/runs/deepflame_c2h4_smoke/c2h4_stock_baseline_np8_gpu_stocksrc`

And the learned path is clearly active, not bypassed.

The reported learned active-set counts remain nonzero and slowly increase over the run, for example:
- `2e-7`: `33508`
- `1e-6`: `33912`
- `2e-6`: `34197`
- `5e-6`: `34730`

Also:
- `solver.err` remains empty

## Why this matters

This is the first C2H4 runtime baseline in this phase that looks like a trustworthy DeepFlame reference rather than just a debugging artifact.

That means the project can now stop asking only:
- “can the stock C2H4 case run?”

and start asking the more important next question:
- “what learned replacement path has a realistic chance of surviving this same stock-style runtime target?”

## What comes next

The baseline target for C2H4 is now concrete:
- **run a learned model in this simulation and survive to `5e-6`**

So the next work should pivot toward:
- case-aligned C2H4 training data and contracts
- learned-model integration design
- the first DeepFlame-facing surrogate that can be tested against this `5e-6` stock runtime baseline

Detailed findings and run summaries are recorded in the findings pages.
