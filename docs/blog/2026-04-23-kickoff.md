# Kickoff: EFNO replication program starts with paper ingestion and baseline mapping

_Date: 2026-04-23_

Today I started the EFNO replication-and-extension program in earnest.

## What was done

- read the project operating contract in `AGENTS.md`
- surveyed the workspace, docs site, paper library, DeepFlame tree, DFODE-kit tree, and target CFD cases
- confirmed the two target coupled cases under DeepFlame:
  - `twoD_HIT_flame/H2`
  - `twoD_HIT_flame/C2H4`

- extracted the EFNO paper into reusable plain-text artifacts under:
  - `/root/workspace/artifacts/papers/efno_weng_2025/`
- wrote the first program plan and paper-spec notes

## Early findings

- the paper text extracts cleanly enough for structured implementation notes
- the paper is conceptually clear on the big ideas:
  - FNO backbone
  - Box-Cox transform
  - physics-informed conservation losses
  - imbalance-aware weighting
  - autoregressive rollout evaluation
- several practical hyperparameters are not obvious from the paper text alone:
  - Fourier mode count
  - latent width
  - optimizer/schedule
  - batch size
  - numeric weights on the combined loss terms
- the balanced-loss section is especially important: the paper discusses focal loss, but the apparent final experimental form is a quartile-based weighted MSE-like loss

## Important alignment note

The paper's final turbulent benchmark is a 3D NH3/H2 LES case, while the target project CFD cases are DeepFlame's bundled 2D HIT-flame H2 and C2H4 examples. So the program will reproduce first, then adapt carefully, with all deviations documented.
