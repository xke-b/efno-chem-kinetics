# External papers sharpen the next benchmark extension: move from species-only deltas toward thermochemical-state learning

_Date: 2026-04-23_

## Why this was the next useful step

The EFNO reproduction is still blocked by an important fidelity gap: our current DFODE-kit experiments are learning only a species-delta target, while the EFNO paper and nearby related work are much closer to **full thermochemical-state evolution**.

So the next useful step was to combine:
1. external-paper evidence from web/OpenAlex search, and
2. inspection of the current local code path,

to answer a concrete question:

> Is there enough evidence to justify making `T + species` prediction the next benchmark extension?

The answer is now **yes**.

## External evidence gathered

### 1. H2 operator-learning paper
Source found via Exa:
- page: `https://osta.hanyang.ac.kr/publications/EAAI_JunhyongJo/`
- listed DOI on the page: `10.1016/j.engappai.2024.109704`

Important details from the abstract/page:
- 0D **constant-volume** hydrogen reactor
- **11 thermochemical variables** are modeled
  - `9` species
  - `temperature`
  - `pressure`
- variables span up to **11 orders of magnitude**
- uses **data-augmented physics-informed operator learning**
- explicitly studies how a small amount of pre-labeled data can improve training efficiency

Why this matters here:
- it directly reinforces that **joint thermochemical prediction** is normal and useful in this literature
- it reinforces that **scale disparity** is a central design issue
- it also suggests that `pressure` handling should be documented explicitly, even in 0D settings

### 2. NH3/H2 turbulent-combustion acceleration paper
Sources:
- Exa result page: `https://discovery.ucl.ac.uk/id/eprint/10210155/`
- downloaded open PDF: `/root/workspace/artifacts/papers/nh3_h2_wu_2025/wu_liang_luo_2025_nh3_h2_turbulent_combustion.pdf`
- extracted text: `/root/workspace/artifacts/papers/nh3_h2_wu_2025/extracted/full_text.txt`

Important details from the paper:
- variable-time-step DNN chemistry model
- training manifold built from **1D unity-Lewis-number laminar premixed flames**
- broadened using **continuous randomization**
- uses **mass conservation** and **heat-release-rate similarity** constraints in training
- labels are generated by advancing perturbed states over **random time steps from `1e-8` to `1e-6 s`**
- performs **a-posteriori validation** on:
  - planar turbulent premixed flames
  - temporally evolving jet flames
- chosen network for deployment: **4 hidden layers, 800 neurons**
- reported speedups:
  - chemistry acceleration about `7x` on CPU, `30x` on GPU
  - overall simulation acceleration about `5x` on CPU, `10x` on GPU

Most relevant training-data details extracted from the paper:
- 77 transient 1D flames
- equivalence ratio range: `0.5` to `1.5`
- H2 blend fraction range: `0` to `0.30`
- 100 reaction-progress-variable samples per saved flame state
- `7.7 million` initial states and `7.7 million` perturbed states
- mass fractions randomized multiplicatively with many small random factors rather than a single coarse perturbation
- perturbed states filtered with **HRR** and **net-production-rate** criteria to reject unphysical samples
- uses **Box-Cox transform** with `lambda = 0.1` for mass fractions
- DNN input includes `[T, p, BCT(Y), dt]`
- DNN target is `[BCT(Y(t+dt)) - BCT(Y(t))]`

Why this matters here:
- it strongly supports the importance of:
  - manifold-based data construction
  - explicit randomization/augmentation
  - variable-`dt` training
  - a-posteriori validation under unseen turbulent conditions
- it also shows a subtle but important point:
  - even when the network output is expressed in transformed species space, the overall training/evaluation contract is still built around the broader **thermochemical state**

## Current local code reality

### Good news: the dataset generator already contains the full state
`/root/workspace/scripts/generate_h2_autoignition_pairs.py` stores rows as:
- `[T, P, Y_1, ..., Y_ns, T_next, P_next, Y_1_next, ..., Y_ns_next]`

So the data path is already close to what we need.

### Current limitation: the DFODE-kit trainer throws away most of that target information
In `/opt/src/DFODE-kit/dfode_kit/training/train.py`:
- features are built from `T`, `P`, and transformed species
- but labels are currently
  - `BCT(species_next[:-1]) - BCT(species_now[:-1])`
- meaning the learned target is still **species-only**, and not even the full thermochemical target

This is the key local fidelity gap.

## New conclusion

The evidence now points to a more specific next benchmark step:

### Recommended next extension
Prototype a **thermochemical target benchmark** that predicts at least:
- `T_next` or `delta_T`
- all species next-state information

Pressure can remain fixed or carried through explicitly depending on the reactor assumption, but the choice should be documented clearly.

## Why this should come before more architecture churn
Right now, changing architectures alone is unlikely to answer the important reproduction question, because the learning contract itself is still misaligned with the paper neighborhood.

The stronger near-term order of operations is:
1. improve target fidelity
2. preserve holdout-by-trajectory evaluation
3. keep rollout and physical-consistency checks
4. then revisit whether EFNO-style constraints help

## Practical implications for the next implementation step

A small but meaningful next code step would be:
1. add an alternative training path that predicts `delta_T` alongside species targets
2. update the evaluator to report temperature errors as well as species/element/mass metrics
3. compare this against the current species-only baseline on the same H2 holdout split

That would produce a much more paper-relevant result than adding another model backbone right now.

## Bottom line

The most useful next reproduction step is no longer just "tune EFNO-style loss more" in the abstract.

It is:

> extend the current offline benchmark from species-only deltas toward a thermochemical-state target, while keeping the harder holdout evaluation we already established.
